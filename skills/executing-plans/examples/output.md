# 示例输出

## 期望行为

- 先加载并批判性审查书面计划，再开始执行。
- 如果计划里有依赖缺口、验证条件含糊或环境假设未说明，先向用户提出，而不是直接猜。
- 在当前会话中顺序推进任务，使用 TodoWrite 跟踪状态，并在检查点回顾方向是否偏离。
- 当环境没有实现型子代理或任务耦合较强时，作为主执行路径稳定推进。
- 完成后输出固定执行报告；只有用户明确要求提交时，才衔接 `git-commit` 或常规 git 提交流程。

## 固定回复骨架

```markdown
执行已完成。

**计划：** `docs/superpowers/plans/<filename>.md`
**执行结论：** `全部完成` | `部分完成` | `阻塞`
**任务进度：** `<completed>/<total>`

**已完成任务：**
- [任务编号 + 结果]

**验证结果：**
- [测试 / lint / 构建 / 手动检查]

**偏离计划：**
- [若无则写 `- 无`]

**阻塞或风险：**
- [若无则写 `- 无`]

**下一步：**
- [若用户要求提交则说明提交路径，否则说明等待下一步指令]
```

## 不期望行为

- 在没有书面计划时直接开始执行。
- 明明更适合 `subagent-driven-development`，却忽略环境能力和任务独立性直接内联执行。
- 遇到阻塞或计划缺陷时硬闯过去。
- 未经用户要求强行进入额外的旧收尾 skill 或自动提交。
- 只说“已执行完”而不给任务进度、验证结果和阻塞信息。
