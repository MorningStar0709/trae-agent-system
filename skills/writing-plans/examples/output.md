# 示例输出

## 期望行为

- 基于清晰规格或已批准设计生成可执行实现计划，而不是直接开始编码。
- 在计划中明确目标、架构、技术栈、文件结构、任务边界、验证路径和完成信号。
- 任务拆分包含前置依赖、可执行步骤、验证命令、预期结果和 commit 节奏。
- 最终回复使用固定交接骨架，先给出默认推荐，再给出备选方式。
- 如果当前环境不支持实现型子代理，明确推荐 `executing-plans`，不引用不存在的旧能力名称。

## 固定回复骨架

```markdown
计划已完成并保存到 `docs/superpowers/plans/<filename>.md`。

**计划摘要：**
- 目标：[一句话]
- 任务数：[N]
- 关键文件：[2-5 个最重要的路径]

**推荐执行方式：** `subagent-driven-development` | `executing-plans`
**推荐理由：** [基于环境能力和任务耦合度的 1-2 句话]
**备选方式：** `subagent-driven-development` | `executing-plans`
**下一步：** [直接说明应调用哪个技能继续推进]
```

## 不期望行为

- 对需求仍模糊的问题直接输出实现计划，而不是先交给 `brainstorming`。
- 对简单小改动也生成冗长计划。
- 在用户已经有现成计划时重复规划，而不是转去执行。
- 输出只有高层摘要，没有文件、测试、验证、前置依赖或完成信号。
- 把执行路由完全留给用户自己判断，而不先给出默认推荐。
