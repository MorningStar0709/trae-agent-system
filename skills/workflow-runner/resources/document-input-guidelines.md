# 文档型输入约定

当工作流输入中出现如下字段时，`workflow-runner` 应优先把它们识别为“需要读取的文档”，而不是普通字符串：

- `*_file`
- `*_path`
- `prd_file`
- `spec_file`
- `doc_file`
- `brief_file`

## 执行规则

- 如果值看起来是本地文件路径，先用 `Read` 读取文件内容。
- 在角色执行前，把文档的关键内容与原始任务一起提供给角色。
- 不要只把路径字面量传给角色然后假设角色会自己读取。
- 如果文档过长，先提炼摘要，但要保留关键章节、目标、约束、验收标准等高价值信息。

## 失败处理

- 路径不存在：报告具体路径并请求用户提供有效文件。
- 文件可读但内容为空：说明无法基于空文档执行高质量评审。
- 文档过大：先提炼摘要，再明确说明本次基于摘要而非全文执行。
