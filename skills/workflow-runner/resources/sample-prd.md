# 智能筛选规则配置 PRD

## 背景

运营团队希望在后台新增“智能筛选规则”能力，允许非技术同学为活动页面配置筛选条件，从而在不同场景下展示不同的内容模块。

## 目标

- 支持按用户标签、地区、设备类型配置筛选条件
- 支持多条件组合，并允许保存为模板
- 支持预览某个用户命中哪些规则

## 非目标

- 本期不支持基于实时行为流的动态计算
- 本期不支持复杂脚本表达式

## 用户故事

- 作为运营，我希望创建一个规则，把“华东地区且 iOS 用户”命中到 A 模块
- 作为运营，我希望复制已有规则并微调，而不是每次从零创建
- 作为客服，我希望输入一个用户 ID 后查看其命中的规则，便于排查问题

## 功能要求

### 规则编辑

- 支持新增、编辑、删除规则
- 每个规则包含名称、优先级、条件列表、命中模块
- 条件之间默认使用 AND，暂不支持 OR 组

### 模板能力

- 支持把规则保存为模板
- 新建规则时可从模板复制

### 预览能力

- 输入用户 ID 后，返回命中的规则列表和最终命中的模块

## 约束

- 首期必须沿用现有用户标签服务，不新增标签存储系统
- 预览接口响应时间目标小于 500ms
- 前后端团队本期总开发资源约 2 人周

## 验收标准

- 运营可以创建至少 10 条规则并正常保存
- 预览功能对 3 类典型用户可返回稳定结果
- 没有配置规则时，页面继续走默认模块逻辑
